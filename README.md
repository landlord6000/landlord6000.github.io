# Как писать черновик отчёта

Просто пишешь текст сплошным потоком. В местах, где должно быть фото —
отдельная строка `фотоNN подпись`.

```
Вышли рано утром, дошли до реки.
фото00 Утренний туман над рекой
фото01 Мост через реку

Дальше тропа пошла в горку, шли долго и тяжело.
фото10 Подъём в горку
```

## Правила

- `NN` — двузначный номер: первая цифра — номер смыслового блока
  (день/точка/эпизод) по счёту от начала (0, 1, 2...), вторая —
  номер фото внутри блока (0, 1, 2...).
- Первое фото в блоке — всегда `N0` (`00`, `20`, `60`), даже если оно
  там одно.
- Подпись пишется всегда, сразу после номера через пробел (без
  двоеточия, без скобок).
- Несколько `фотоNN` подряд без текста между ними — это одна галерея.
- Заголовки блоков (день/точка) можно не расставлять руками — просто
  начинай новый смысловой кусок с новой строки/абзаца, Клод сам
  разобьёт на блоки и присвоит номера, либо укажи явно, если хочешь
  конкретную нумерацию точек (X1, X2...) или подпись дня («День первый»).

## Что нужно приготовить кроме текста

- Название отчёта, тип похода, дата (месяц.год), километраж — одной
  строкой, для подзаголовка.
- Ссылка на трек (nakarte.me или аналог).
- Сами фото — заранее разложенные по `photos/NN.jpg` и `thumbs/NN.jpg`
  в папке `travel/<slug>/`. Клод разметку сгенерит, но сами файлы фото
  кладёшь ты (уменьшать/переименовывать можно скриптом или руками).

## Что Клод сделает сам

- Разобьёт текст на блоки, подставит `<img>` с нужными путями и alt
  из твоих подписей.
- Соберёт galleries (одиночное фото — широким кадром с подписью снизу,
  несколько подряд — сеткой).
- Создаст `travel/<slug>/<slug>.html` по общему шаблону (использует
  общие `assets/report.css` и `assets/report.js` — их трогать не надо).
- Добавит новый отчёт первым пунктом в `index.html` и первой записью
  в список `REPORTS` внутри `assets/report.js` (это нужно для стрелок
  «предыдущий/следующий отчёт» внизу страницы).

Полная инструкция для Клода — в `CLAUDE.md`, эта памятка просто
шпаргалка по формату текста для тебя.
